The dashboards settings allow configuration of what files are presented and the URLs to use to present them to users.

Public Dashboards

Public dashboards are accessible to anyone, even users not logged in. Be aware that to access any data within Mango there must be an active User session for that page.

Public URL prefix

This setting is the base URL for users to use to access public files. This value must be postfixed with a slash for the mapping to work.

Public files location

This setting is the directory within the module that contains the public files. This value must be postfixed with a slash for the mapping to work.

Private Dashboards

Private dashboards are accessible only to logged in users. If a user that is not logged in tries to access the page they are either directed to the Mango login page, or if a value is set in the login page property then they are directed to that page.

Private URL prefix

This setting is the base URL for users to use to access private files. This value must be postfixed with a slash for the mapping to work.

Private files location

This setting is the directory within the module that contains the private files. This value must be postfixed with a slash for the mapping to work.

Icon Settings

The icon is the image shown on the Mango Toolbar. The icon location is the path within the module to the icon file, do not prefix with a slash. The icon destination is the url where the user will be directed when clicking on the icon.

Login page

This setting allows the user to override the login page for Mango. For example to direct users to the public/login.htm page the value would be /public-dashboards/login.htm. Use with caution as all non-logged in users will be directed to this page so if it is broken no one will be able to login until this setting is changed. If this happens just run this SQL on the database, 'DELETE dashboards.login.page from systemSettings' to remove the bad mapping.

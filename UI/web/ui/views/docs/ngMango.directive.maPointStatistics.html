maPointStatistics
directive in module ngMango

Description

<ma-point-statistics point="myPoint" from="from" to="to" statistics="statsObj"></ma-point-statistics>

  • The <ma-point-statistics> directive provides access to historical stats over a time range on a data point.
  • The object returned by the statistics attribute contains first, last, minimum, maximum, average, integral, sum, & count properties. Each of these will have a value and a timestamp.
  • If you are interested only in the change in a value between two times you can add the optional first-last="true" attribute to only return the first and last values, then simply calculate the difference with {{ statsObj.last.value - statsObj.first.value }}.

Usage

<ma-point-statistics point="myPoint" from="from" to="to" statistics="statsObj">
</ma-point-statistics>
The average for the period is {{ statsObj.average.value }} at {{ statsObj.average.timestamp | maMoment:'format':'lll' }}

Parameters

ParamTypeDetails
pointobject

Inputs a point object from <ma-point-list>.

fromobject

Sets the starting time for the time range which is used in the statistical query.

toobject

Sets the ending time for the time range which is used in the statistical query.

statisticsobject

Outputs the object containg the statistcal information. The object will contain the following properties:

  • first
  • last
  • minimum
  • maximum
  • average
  • integral
  • sum
  • count
Each of these will have a value and a timestamp.

first-last
(optional)
boolean

If you are only interested in calculating the delta value, setting this to true will run a more efficient query and only return first and last properties on the statistics object.

points
(optional)
array

Alternatively you can input an array of points from <ma-point-query>. If used the statistics object will output an array.

point-xid
(optional)
string

Alternatively you can pass in the xid of a point to use.

date-format
(optional)
string

If you are passing in to/from as strings, then you must specify the moment.js format for parsing the values.

timeout
(optional)
number

If provided you can set the timeout (in milliseconds) on the querying of of the statistical provider. If not supplied the Mango system default timeout will be used.

rendered
(optional)
boolean

(default true) Return the statistics as a rendered value rather than a number

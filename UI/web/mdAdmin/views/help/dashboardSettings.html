The Dashboard Settings page can be used to change the Theme hues (colors) of the dashboard, load a different Logo, and set the Code coloring:

If you would like to use your own logo, you should put it inside your Mango/overrides so that it persists through module updates. For instance, create Mango/overrides/web/modules/ui/web/images if you haven't already and place your logo image in there. You can then reference the logo using /modules/ui/web/images/myLogo.png

If you choose the userTheme option in the Choose theme select, you can choose from the 20 Material Design Color Palettes to use within the Mango UI.

Per the Material Design specification, you should choose one color palette for each of the three color intentions plus one for the background palette.

  • Primary palette - Used to represent primary interface elements.
  • Accent palette - Used to represent secondary interface elements.
  • Warning palette - Used to represent interface elements that the user should be careful of.

The Mango UI will automatically update it's interface elements to use the color palettes, so remember to save before navigating the page or it will change back! Advanced users can look into Mango/web/modules/ui/web/mdAdmin/dashboardSettings.json if they wish to take color customization to the next level.

You can turn on Dark theme on any theme by checking the Dark theme option. A dark version of the background palette will be used and white text will display on top for maximum contrast.

Additionally you can set the Code coloring theme to be used across the dashboard. Choose a theme from the dropdown and you will see a live preview of the code coloring below. Once saved, the Code coloring will be used in Edit pages, Play area, Examples, and API Docs.

The Dashboard Settings page can be used to change the Theme Colors of the dashboard, swap out the Logo with your own, and set the Code Coloring.

If you would like to use your own logo you should put it in the image folder in your overrides. Create a new folder at ${MA_HOME}/overrides/web/modules/dashboards/web/images if you haven't already and place your logo image in there. You can then reference the logo using /modules/dashboards/web/images/myLogo.png

If you choose the User Theme option you can choose from the 20 Material Design Color Palettes to use with the Mango UI.

Per Material Design Spec. you should choose one Color Palette for each of the Three Color Intentions plus one for the Background.

  • Primary Palette - Used to represent primary interface elements.
  • Accent Palette - Used to represent secondary interface elements.
  • Warning Palette - Used to represent interface elements that the user should be careful of.

The Mango UI will automatically update it's interface elements to use the Color Palettes across the UI.

You can turn on Dark Mode on any theme by checking the Dark Theme option. A dark version of the background palette will be used and white text will display on top for maximum contrast.

Additionally you can set the Code Coloring Theme to be used across the dashboard. Choose a theme from the dropdown and you will see a live preview of the code coloring below. Once saved, the Code Coloring Theme will be used in the Page Editor, Play Area, Demos, and API Docs.

Custom pages can be created from within the app's CMS environment, using our built in Code Editor and UI Component Library to create and link new pages to the app. Mango's HTML Web Components combined with Angular templating logic allows you to create simple or complex dynamic pages that pull data from Mango's database and visualize the data as responsive interface elements, charts and tables. Interactive Demos and Examples can be found under the Examples menu section. Full documentation of all parameters of each component can be found under API Docs.


The Page Editor

The Page Editor can be found under Settings > Edit Pages in the menu. You can use this tool to manage all of your custom pages.

The inputs above the button row allow you to choose a pre-existing page to edit, update the name of a page, and set the page's permissions. Read / Edit Permissions can be used to prevent access to a page.

The first button will Save your changes to the page you are currently editing. The Revert button will reset the code editor to the last saved state. The Delete button will remove a page from the sytem after confirming. The New Page button will set-up a blank page for you to work with. Be sure to Save changes to the previous page first.

Before you can Save a new page you will need to give it a Name.

You can Toggle on and off the Editor Pane and Preview Pane allowing you to view just one pane or both in a side-by-side view.



Editing Code

You will enter HTML into the code editor on the left. If you prefer you can work in your desktop code editor of choice and Copy & Paste your code into the Editor Pane.



Previewing the Page

You will see a live preview of your page in the Preview Pane on the right as you enter code. At any time you can Save and View the page in full screen or even from a different browser or mobile device.

The View Page button will open the last saved state of the page in full-screen mode in a new tab. This is the same URL that you can use to share the page with somebody and will be used by the page's Menu Item.


Starting from an Example or Template

A quick way to get started is to click the Copy Markup to New Custom Page link from any of the example pages. This will create a new page with the code copied into it.

If you are looking to start with a pre-made layout you can also copy one of the Adaptive Layout templates as a starting point. Click the New Page From Layout button to create a new page using the layout shown. These examples give you a predefined responsive layout that you can adapt to your needs.



Adding Your Own Images

If you would like to include your own images on a page you can put them in an image folder in your overrides. Create a new folder at ${MA_HOME}/overrides/web/modules/dashboards/web/images and place your image files in there. You can then insert the image onto a page using: <img ng-src="/modules/dashboards/web/images/myImage.png" alt="My Image"/>


Permissions

By default, all pages that are created can be viewed by all users and require the user to have the edit-pages permission to edit them. If you want to restrict the pages to be viewable only by admin users, change the read permission to superadmin.

Users with the edit-menus permission can edit the menus. When editing the menu items you can set a permission for each menu item, if the user does not have this permission the menu item will not be shown.


Adding a Menu Item

The Add/Edit Menu Item button will open up the Edit Menu Item dialog.

From here you will give the menu item a Parent Item, URL, Label, and Menu Icon. The entire menu is completely customizable from the Menu Editor under Settings > Edit Menu in the menu. You can Hide/Show menu items, Edit menu item Labels and Icons, Reorder menu items, add New menu items and Delete custom menu items. For more information see the Menu Help Page.
